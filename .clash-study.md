## Proxy Group 类型（Clash 支持）

| 类型 | 行为 | 自动选优 |
|------|------|----------|
| `Selector` | 手动选节点 | ❌ |
| `URLTest` | 自动选最快（定时测速） | ✅ |
| `Fallback` | 节点故障自动切换备选 | ✅ |
| `LoadBalance` | 轮询/哈希分发 | ✅ |
| `Relay` | 链式代理（多跳） | ❌ |

## 自动选优机制

### URLTest（现有）
```yaml
- name: 🚀 AUTO-全球
  type: url-test
  url: https://www.gstatic.com/generate_204
  interval: 300  # 每5分钟测速
  tolerance: 100
  health-check: true
```
- 每5分钟对所有节点测速
- 选延迟最低的节点
- Clash 自动执行，无需脚本干预

### GLOBAL 组关键修复

**修复：** `GLOBAL → 🚀 AUTO-全球`（通过 Clash API 切换）
**效果：** 系统所有 HTTP/HTTPS 流量自动走最优节点

## 进阶自动选优方案

### 方案1：Fallback 组（已配置但未启用）
当主节点挂了，自动切换备用节点。
```yaml
- name: 🛡️ AUTO-全球-Fallback
  type: fallback
  proxies:
    - 🚀 AUTO-全球
    - 🔧 AUTO-HK
    - 🔧 AUTO-JP
  health-check: true
```

### 方案2：LoadBalance（可提升吞吐量）
```yaml
- name: ⚖️ 负载均衡
  type: load-balance
  strategy: round-robin  # 或 consistent-hashing
  proxies:
    - 🚀 AUTO-全球
    - 🔧 AUTO-HK
```

### 方案3：TUN 模式（系统级代理）
接管全部系统流量，不依赖环境变量。
需要：Clash Premium 或 meta 版本，内核模块支持。
